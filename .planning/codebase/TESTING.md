# Testing

## Frameworks & Structure

### Frontend (`coc_frontend`)
- Currently, there is no configured testing framework (e.g., Vitest, Jest) based on the `package.json` scripts.
- The project relies on TypeScript for type checking and ESLint for static analysis to ensure code quality.

### Smart Contracts (`coc_web3`)
- There are no unit or integration tests present in the `coc_web3/tests/` directory (the folder does not exist).
- For local or testnet validation, developers will need to invoke the contracts manually via the `sui client` or frontend integration.

## Coverage
- Test coverage metrics are currently not available.
